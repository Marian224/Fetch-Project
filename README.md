# Fetch Project (JSONPlaceholder Dashboard)

Простий та інтерактивний веб-додаток (веб-панель) для отримання та відображення демонстраційних даних із безкоштовного API **JSONPlaceholder** за допомогою бібліотеки **Axios**.

## 🚀 Функціонал

* **Пагінація (завантаження додаткових даних):** При натисканні на кнопку кожна секція підвантажує по 10 нових елементів порціями (`_limit=10`), не перезаписуючи старі. Текст кнопки динамічно змінюється на "Fetch more...".
* **Підтримка 6 типів ресурсів:**
    * **Posts (Пости):** Показ заголовка, ID поста, ID автора та тексту.
    * **Comments (Коментарі):** Показ імені, ID, тексту коментаря та пошти автора.
    * **Albums (Альбоми):** Показ назви альбому та ID власника.
    * **Photos (Фото):** Завантаження галереї. Замість неробочих стандартних плейсхолдерів інтегровано сервіс **Picsum Photos**.
    * **Users (Користувачі):** Повна інформація про користувачів (ім'я, нікнейм, телефон, сайт тощо).
    * **Todos (Завдання):** Візуальний розподіл завдань на виконані (із позначкою ✓ та напівпрозорим фоном) та невиконані.
* **Інтерактивне модальне вікно:** При кліку на будь-яку мініатюру фотографії відкривається повноекранне модальне вікно із зображенням у високій якості.

---

## 🛠️ Технологічний стек

* **HTML5** — Структура сторінки та модальних вікон.
* **CSS3** — Гнучка сітка (Flexbox), кастомні рамки для кожної картки, ефекти збільшення (hover) для фото та адаптивний дизайн модального вікна.
* **JavaScript (ES6+)** — Асинхронні запити (`async/await`), маніпуляції з DOM та динамічне створення розмітки.
* **Axios (CDN)** — HTTP-клієнт для надсилання запитів на сервер.
* **JSONPlaceholder API** — Джерело фейкових даних для тестування.

## ⚙️ Як запустити проект

Оскільки проект написаний на чистому JavaScript (Vanilla JS) і використовує CDN для Axios, вам не потрібно нічого встановлювати через npm.

1. **Завантажте файли** (`index.html`, `style.css`, `script.js`) в одну папку на комп'ютері.
2. **Запустіть проект** одним із способів:
* **Варіант А (Рекомендовано):** Відкрийте проект за допомогою розширення **Live Server** у Visual Studio Code (клікніть правою кнопкою миші на `index.html` -> *Open with Live Server*).
* **Варіант Б:** Просто двічі клацніть на файл `index.html`, щоб відкрити його у будь-якому браузері.


3. Переконайтеся, що у вас є **підключення до інтернету**, оскільки Axios та дані завантажуються з мережі.

## 💻 Особливості реалізації в коді

### Стилізація карток за кольорами:

Для кращої візуальної орієнтації кожна категорія має свій колір бордера:

* 🟢 Пости — **Зелений** (`green`)
* 🔵 Коментарі — **Синій** (`blue`)
* 🟣 Альбоми — **Фіолетовий** (`purple`)
* 🟠 Фото — **Помаранчевий** (`orange`)
* 🔴 Користувачі — **Червоний** (`red`)
* 🔵 Завдання — **Бірюзовий** (`teal`)

### Робота з фото (Modal & Picsum):

Через те, що стандартні посилання на фото в JSONPlaceholder часто бувають недоступними, у `script.js` реалізовано підміну на стабільні випадкові фото з `picsum.photos`:

```javascript
const workingThumbnail = `https://picsum.photos/150?random=${id}`;
const workingFull = `https://picsum.photos/600?random=${id}`;

```

Також додано механізм делегування подій (через невеликий `setTimeout`), який автоматично навішує слухач кліку на нові завантажені фото для відкриття модального вікна.
